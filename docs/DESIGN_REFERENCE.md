## 🎯 开发时必须对照

### 每个组件开发完成后：

1. **打开对应的设计稿图片**
2. **将浏览器窗口与设计稿并排放置**
3. **逐像素对比检查**：
   - 颜色是否完全一致？
   - 圆角大小是否一致？
   - 间距是否一致？
   - 字体大小是否一致？
   - 布局结构是否一致？

### 验收标准

**设计稿与实现放在一起，用户看不出任何差异。**

## 🎨 精确颜色值（从设计稿提取）

### 主要颜色

```css
/* 背景 */
--bg-cream: #F5F1E8;        /* 奶油白 */
--bg-white: #FFFFFF;        /* 纯白（卡片） */
--bg-gray: #F8F8F8;         /* 浅灰 */

/* 主色调 */
--brown-warm: #8B6F47;      /* 暖棕色（标签、按钮） */
--brown-dark: #5C4A37;      /* 深棕色（COOK NOW、AI卡片） */

/* 强调色 */
--orange-accent: #E8A87C;   /* 橙色（状态检查图标） */

/* 文字 */
--text-dark: #333333;       /* 深色文字 */
--text-gray: #666666;       /* 灰色文字 */
--text-light: #999999;      /* 浅灰文字 */

/* 全屏模式 */
--fullscreen-bg: #1A1A1A;   /* 黑色背景 */
--fullscreen-text: #FFFFFF; /* 白色文字 */
```

## 💡 开发建议

### 1. 使用设计稿作为背景对比

```css
/* 开发时临时使用 */
.debug-overlay {
  background-image: url('/docs/prd-images/image2.png');
  background-size: contain;
  background-repeat: no-repeat;
  opacity: 0.3; /* 半透明叠加 */
  pointer-events: none;
}
```

### 2. 使用浏览器开发工具测量

- 使用 Chrome DevTools 的 Ruler
- 对比间距、字号、颜色
- 检查圆角、阴影

### 3. 截图对比

- 实现后截图
- 与设计稿并排对比
- 用 Photoshop/Figma 叠加检查

## 🚨 常见错误（避免）

❌ **错误示范**：
- "这个颜色差不多就行"  → ✅ 必须精确匹配
- "圆角看起来差不多"    → ✅ 必须用设计稿的值
- "间距大概这样吧"      → ✅ 必须测量精确值
- "字体大小差不多"      → ✅ 必须完全一致

### 组件开发完成后：
